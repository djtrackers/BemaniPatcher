Update your version of SpiceTools

Importing patches from a remote URL is available as of the pre-release 24-04-30 version.

Due to changes in the filename format, you will need to upgrade to build spice2x-24-04-30_r0001.zip or newer to use the patches available here. To install this version, download the asset from the link below, then extract the new spice.exe, spice64.exe, and spicecfg.exe files from the .zip file over your existing installation.

Download pre-release version

Import patches from URL

Patches can be imported and managed before the game is launched, using the standalone spicecfg.exe configurator tool from the "Patches" tab, or while the game is running using the in-game overlay.

Press the Import from URL button and enter the URL below into the text field: 

https://djtrackers.github.io/BemaniPatcher

Finally, press the Import button to download and save a list of patches for your game.

Configure the imported patches

If all went well, a list of patches should now be available for configuring. These are saved locally to your patches directory as a JSON file, which will be loaded automatically each time you start the game.

Each patch JSON file retrieved from URL corresponds to a specific version of the game. If you ever upgrade to a newer version of the same game, you will need to repeat the steps above.

Be aware that not every game version is currently supported by this mirror repository. The compatibility list below tracks which versions supported by the upstream repository are actively being converted into spice2x format. If you have a version that is not yet supported, please consider submitting metadata for it.

If you would like to contribute patches for a brand new game or specific version of a game, send a pull request to the upstream BemaniPatcher project. Be sure to check the README to know what is allowed first!
